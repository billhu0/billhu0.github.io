[Lecture Notes] ShanghaiTech CS252 Cryptography

Lec 1

Information security

Information security: The protection of information and information systems from unauthorized access, use, disclosure, disruption, modification, or destruction in order to provide confidentiality, integrity, and availability.

  • Confidentiality: the property that sensitive information is not disclosed to unauthorized individuals, entities or processes.

  • Integrity: the property that sensitive data has not been modified or deleted in an unauthorized and undetected manner.

  • Authentication: the process of estabilishing confidence in the identity of users or information systems.

  • Non-Repudiation: assurance that the sender of information is provided with proof of delivery and the recipient is provided with proof of the sender’s identity, so neither can later deny having processed the information.

Lec 2

Private-key Encryption (PrivKE)

  • Gen\mathrm{Gen}Gen = key generation, Enc\mathrm{Enc}Enc = encryption, Dec\mathrm{Dec}Dec = decryption
  • kkk = secret key, mmm = plaintext (message), ccc = ciphertext
  • K\mathcal{K}K = key space, M\mathcal{M}M = plaintext space, C\mathcal{C}C = ciphertext space
  • PrivKE=(Gen,Enc,Dec)+M\mathrm{PrivKE} = (\mathrm{Gen, Enc, Dec}) + \mathcal{M}PrivKE=(Gen,Enc,Dec)+M
    • Correctness: Dec(k,Enc(k,m))=m\mathrm{Dec(k, \mathrm{Enc}(k, m)) = m}Dec(k,Enc(k,m))=m
    • Security: the adversary should not be able to learn mmm from ccc

Kerchkhoff’s Principle

Principle: The cipher method must not be required to be secret, and it must be able to fall into the hands of the enemy without inconvenience. 加密算法应该公开.

Shift Cipher

Scheme: ci=mi+kc_i = m_i + kci​=mi​+k

Brute-force attack: try all possible secret keys

Sufficient Key Space Principle: key space ∣K∣|\mathcal{K}|∣K∣ should be large enough

Substitution Cipher

Scheme: ci=σ(mi)c_i = \sigma(m_i)ci​=σ(mi​)

  • Key space is large for brute-force attack.

  • Still not secure! Can be broken letter frequency analysis. 可通过字母频率破解.

Vigenere Cipher

Scheme: cic_ici​ = the letter obtained by shifting mim_imi​ forward k(i mod t)+1k_{(i\ \mathrm{mod}\ t) + 1}k(i mod t)+1​ positions

  • Key space is infinite in theory. In addition, same letter in plaintext can be mapped into different letters in ciphertext

  • Still not secure!

    • Kasiski’s method: two identical segments of plaintext may be encrypted to same ciphertext if their distance is a multiple of key length ttt.

      Search for pairs of identical segments → record distances → calculate GCD

      Find the key length ttt and then break ttt shift ciphers.

Lec 3.

Classical & Modern Cryptograohy

  • Classical cryptography: art, not science

    • no formal definition of security
    • no proof of security
    • all historical ciphers have been badly broken (ciphertext ccc → adversary can recover both message mmm and secret key kkk )

  • Modern cryptography: a science

    • Principles: 三个原则
      1. Formal definitions of security
      2. Precise assumptions
      3. Proofs of security

Formal definitions of security

Threat model: defines the power of the adversary

Typical threat models for private-key encryption (COA, KPA, CPA, CCA)

  • Ciphertext-only Attack (COA) 密文攻击
    • Adversary observes ciphertext
    • It tries to determine plaintext
  • Known-Plaintext Attack (KPA) 已知明文攻击
    • Adversary learns pairs of plaintext and ciphertext generated using some key
    • It tries to determine information about plaintext of some other ciphertext produced using the same key.
  • Chosen-Plaintext Attack (CPA) 选择明文攻击
    • Adversary obtains plaintext/ciphertext pairs for plaintexts of its choice (攻击者选择一段明文欺骗A或B帮他加密)
    • It tries to determine plaintext of some other ciphertext produced using the same key.
  • Chosen-ciphertext Attack (CCA) 选择秘闻攻击
    • Adversary obtains plaintext/ciphertext pairs for plaintexts/ciphertexts of its choice. (不仅能CPA还能选择一段密文欺骗A或B帮他解密)
    • It tries to determine plaintext of some other ciphertext produced using the same key

A security definition looks like: a cryptographic scheme for a given task is secure if no adversary of a specific power (指定攻击者的power) can achieve a specified break (指定攻击类型).

Precise Assumptions

Assumption (假设): Statements that are not proven but conjectured to be true 未经证明但猜测为真的命题

E.g The integer factoring problem / discrete logarithm problem is hard.

(The integer factoring problem 大整数分解问题)

Proofs of security

Provable security: If the designed cryptographic scheme can be broken by an adversary, then the underlying assumption is false. (类似于反证法: 假设方案不安全，adv能攻破，推出assumption不成立)

If the scheme is not secure, then a well-known hard problem can be solved in polynomial time.

Lec 4.

Probability

Distributions on K,M,C\mathcal{K,M,C}K,M,C

  • Secret Key KKK: output of Gen\mathrm{Gen}Gen, an r.v. taking values in K\mathcal{K}K
    • Pr[K=k]\mathrm{Pr}[K=k]Pr[K=k], determined by key generation algorithm
    • e.g. KKK is uniform (均匀分布) in shift cipher / substitution cipher
  • Plaintext MMM:
    • Pr[M=m]\mathrm{Pr}[M=m]Pr[M=m], determined by sender’s preference (明文的先验概率)
  • Ciphertext CCC: output of Enc(K,M)\mathrm{Enc}(K,M)Enc(K,M)
    • Pr[C=c]\mathrm{Pr}[C=c]Pr[C=c]

Fundamental assumption: K,M are independent random variables.

基本假设: K M 独立

Perfect Secrecy 完善保密性

  • Definition: Π=(Gen,Enc,Dec)+M\Pi = (\mathrm{Gen, Enc, Dec}) + \mathcal{M}Π=(Gen,Enc,Dec)+M is perfectly secret if

    • ∀\forall∀ distribution MMM, ∀m∈M\forall m\in M∀m∈M, ∀c∈C\forall c\in C∀c∈C with Pr[C=c]>0\mathrm{Pr[C=c]} > 0Pr[C=c]>0,

      Pr[M=m∣C=c]=Pr[M=m]\mathrm{Pr}[M=m|C=c] = \mathrm{Pr}[M=m]Pr[M=m∣C=c]=Pr[M=m]

  • Perfect secrecy requires M,CM,CM,C to be independent

    明文的概率分布和密文概率分布无关

  • For simplicity, assume Pr[C=c]>0\mathrm{Pr[C=c]} > 0Pr[C=c]>0

  • Security guarantee: The adversary learns no additional information about mmm given ccc.

    敌手获得密文c之前和之后，对明文m的认识没有改变（对明文m的后验概率分布和先验概率分布一致）

  • Theorem: Π\PiΠ is perfectly secret if and only if

    • ∀m,m′∈M,∀c∈C\forall m, m' \in \mathcal{M}, \forall c \in \mathcal{C}∀m,m′∈M,∀c∈C

      Pr[Enc(K,m)=c]=Pr[Enc(K,m′)=c]\mathrm{Pr}[\mathrm{Enc}(K,m) = c] = \mathrm{Pr}[\mathrm{Enc}(K,m') = c]Pr[Enc(K,m)=c]=Pr[Enc(K,m′)=c]

One-Time Pad

Scheme:

  • Gen\mathrm{Gen}Gen: k={0,1}nk = {\{0,1\}}^nk={0,1}n
  • Enc\mathrm{Enc}Enc: c=k⊕mc = k \oplus mc=k⊕m (密文是明文和key逐位取异或)
  • Dec\mathrm{Dec}Dec: m=k⊕cm = k \oplus cm=k⊕c

Theorem: one-time pad is perfectly secret.

  • Pr[C=c∣M=m]=Pr[C=c∣M=m′]=2−n\mathrm{Pr}[C=c|M=m] = \mathrm{Pr}[C=c|M=m'] = 2^{-n}Pr[C=c∣M=m]=Pr[C=c∣M=m′]=2−n

Drawbacks:

    1. long secret key (as long as message) 密钥长度与明文长度相同, 太长了

      Kerckhoffs: It must be possible to communicate and remember the key without using written notes, and correspondents must be able to change or modify it as will.

    2. one-time security: same key cannot be used more than once
